Mimosa

A modern browser development toolkit. JavaScript, CSS, and template compilers, linting, optimization, serving, RequireJS support, and Live Reload built right in. Pluggable for authoring your own functionality.

Features

Coding in Seconds

Use Mimosa's command line to build out a skeleton app that is ready to go with your desired meta-languages, a bit of sample code, and an server too! No additional installs or plugins needed to get everything Mimosa has to offer.

Watch Assets

Everything happens when you save your code. Mimosa will keep track of your assets and when you save something, kick off all sorts of awesome.

Immediate Feedback

Code doesn't compile? You'll find out right away. When code breaks Mimosa will write big angry messages to the console and use Growl if you've got it installed.

Asset Serving

Mimosa will serve your assets for you if you'd like, or it will invoke your Express server (or the Express server it gave you) so that you only need a single process running to watch and serve.

Convention First

Mimosa's configuration starts empty. Sane defaults keep you coding instead of figuring out config. Mimosa comes with a set of built-in conventions, and if you stick with those conventions you may never look at or update the configuration.

Configuration Second

Mimosa is heavily configurable if you find yourself needing to vary away from convention, or have your own conventions.

Import Libraries

Mimosa wraps basic Volo add functionality to allow you to easily import known GitHub dependencies right to the directory you want them.

Live Reload

Both the server built into Mimosa and the one it provides you on project creation come with Live Reload ready to go. When a web asset is compiled or a server template is changed, your browser updates immediately.

Optimize + minify, later

Have the best of both worlds. Develop with unminified, uncompressed, individual assets. Flip a switch and develop with minified and compressed assets that have been merged with RequireJS.

Gzipped, Cache-Busted

All your assets will be gzipped, keeping load times as fast as possible, and cache-busted, saving you from dealing with nagging browser cache issues.

Lint it, Hint it

All of your compiled JS and CSS will be run through JSHint and CSSLint which will encourage you to keep your code as idiomatic as possible.

Compilers, Mimosa has them

JavaScript

Mimosa will compile your CoffeeScript, Iced CoffeeScript and LiveScript. We'll make more, just ask.

CSS

Stylus w/nib. SASS w/Compass. LESS. Mimosa won't bother compiling your partials individually, but when your partials change it'll compile whichever files at the root of your CSS tree need compiling.

Micro-Templates

Dust. Handlebars. Hogan. Underscore. LoDash. Jade. Just HTML. Mimosa will pre-compile all of your templates. Not using one of these? Request your favorite be added.

Meta-Languages? Pfft.

Plain JS, Plain CSS, no templates? You're a classic and Mimosa's cool with that. Files that don't need compiling will just be copied, but they'll still get the royal treatment, including linting and RequireJS verification.

AMD? RequireJS? CommonJS?

Dependency Checking

Mimosa will keep track of your dependency graph and alert you right away if you've mis-typed a path, referred to something that doesn't exist, or have a circular dependency.

Shim, Map, Plugin, Path

Yep, yep, yep, yep. It'll check those too. Paths themselves, shims, shim dependencies, map modules and map paths, and plugin paths.

Optimization

If you use RequireJS, you probably know about the optimizer. With the tick of a flag Mimosa will run the optimizer on every successful JavaScript compile or copy. And you won't need to configure a thing.

Also with CommonJS!

Using a Mimosa module, you can code using the CommonJS format and still utilize the offerings of RequireJS, like like path aliasing and mapping, and Mimosa's AMD/RequireJS support, like path verification and application optimization.

Bend Mimosa to Your Will

Pluggable

Mimosa is modular, and makes it easy to plug in external modules to your application and asset workflow. The list of external modules is small for the moment, so write your own, and then tell us about it and we'll mention it on the site.

Discover Modules

Mimosa doesn't do what you need? Maybe someone else built a module to do it. Use Mimosa's module commands to discover plugins in NPM, install ones you want, remove the ones you do not, and list the ones you have.

Jumpstart Module Development

Interested in writing your own module? Mimosa's will get you started with some heavily commented module skeleton code. Wrap your head around Mimosa's workflow concepts, and then just fill in the blanks.
